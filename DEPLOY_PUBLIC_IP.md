# 公网 IP 部署指南

## 上传流程与文件存储说明

### 用户上传时，文件放在哪里？

| 场景 | 存储位置 | 说明 |
|------|----------|------|
| **默认** | `D:\尚品易站图片` | 由 `ASSETS_ROOT` 环境变量或 server 默认值决定 |
| **自定义** | 通过 `ASSETS_ROOT` 指定 | 启动前设置 `ASSETS_ROOT=你的路径` |

**上传路径规则：**

- **普通上传**：文件保存到 `ASSETS_ROOT + 当前浏览目录`。例如当前在「品牌A/产品SKU/产品图片/原图」，则上传到 `D:\尚品易站图片\品牌A\产品SKU\产品图片\原图\`
- **智能分类上传**：按所选产品的文件夹结构自动创建目录（如 `品牌/产品SKU/产品图片/原图`），并自动打标签
- **新建文件夹**：若目标目录不存在，服务端会自动创建（`fs.mkdirSync(..., { recursive: true })`）

### 本机 / 局域网 / 公网 区别

| 部署方式 | 访问方式 | 文件存储 | API 请求 |
|----------|----------|----------|----------|
| **本机** | `http://localhost:43123`（Electron 或浏览器） | 本机磁盘 `ASSETS_ROOT` | 相对路径 `/api/*`，自动解析为 localhost |
| **局域网** | `http://192.168.x.x:43123`（浏览器访问服务器 IP） | 服务器磁盘 `ASSETS_ROOT` | 相对路径 `/api/*`，自动解析为当前访问地址 |
| **公网** | `http://公网IP:43123` | 服务器磁盘 `ASSETS_ROOT` | 同上，需防火墙放行、端口映射 |

**要点：**

- 本机网盘：将 `ASSETS_ROOT` 设为网盘盘符路径（如 `Z:\云资产`），上传文件即存到网盘
- 局域网：服务器在哪台机器，文件就存在哪台机器的 `ASSETS_ROOT`；其他电脑通过浏览器访问该机器 IP
- 公网：同上，文件始终在服务器本地；需确保 43123 端口对外可访问

### 配置 ASSETS_ROOT（可选）

```powershell
# 本机网盘示例
$env:ASSETS_ROOT = "Z:\尚品易站图片"
npm run electron:dev

# 或修改 启动尚品易站云资产.bat，在 npm 前加一行：
# set ASSETS_ROOT=Z:\尚品易站图片
```

### 上传入口

1. **工具栏「上传文件」**：点击后选择文件，上传到当前目录
2. **右键菜单「上传文件」**：在空白处右键，选择上传
3. **拖拽**：将文件拖入窗口，松开后上传到当前目录
4. **智能分类上传**：选择产品与分类，批量上传并自动打标签

## 当前架构说明

- **后端**：Node.js + Express（已监听 `0.0.0.0`，支持外网访问）
- **前端**：React + Tailwind CSS
- **部署**：办公室 Windows 电脑，拥有公网 IP

## 公网访问配置步骤

### 1. 防火墙放行端口

在办公室电脑的 Windows 防火墙中放行 43123 端口：

```powershell
# 管理员 PowerShell
netsh advfirewall firewall add rule name="尚品易站云资产" dir=in action=allow protocol=tcp localport=43123
```

### 2. 路由器端口映射（如需要）

若办公室电脑在路由器/NAT 后，需在路由器中做端口映射：

- 外部端口：43123
- 内部 IP：办公室电脑的局域网 IP（如 192.168.1.100）
- 内部端口：43123

### 3. 启动服务

```bash
npm run build
npm run server
# 或使用 Electron 打包后运行
```

### 4. 访问地址

- 局域网：`http://192.168.x.x:43123`
- 公网：`http://你的公网IP:43123`

## 上传卡住 / 一直显示「上传中」的常见原因

| 原因 | 处理方式 |
|------|----------|
| 网络延迟高 | 已把上传超时改为 5 分钟，大文件需等待 |
| 防火墙拦截 | 检查 43123 端口是否放行 |
| 端口未映射 | 确认路由器已做端口映射 |
| 文件过大 | 单文件限制 500MB，更大需调整 `server.ts` 中 `limits.fileSize` |
| 跨域 / CORS | 当前已启用 CORS，同源访问一般无问题 |

## 关于迁移到 FastAPI 的说明

若计划迁移到 **Python FastAPI**：

- **优势**：异步、流式传输、类型提示、自动文档
- **工作量**：需重写后端 API、鉴权、文件上传等逻辑
- **建议**：当前 Express 已支持公网部署，可先完成公网访问和稳定性验证，再评估是否迁移

当前 Express 已具备：
- 多用户权限（Admin/Editor/Viewer）
- JWT 鉴权
- 文件上传、打标签、缩略图
- 0.0.0.0 监听，支持公网访问
