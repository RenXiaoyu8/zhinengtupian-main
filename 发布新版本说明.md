# 发布新版本（让已装客户端自动更新）

## 流程概览

1. 改版本号 → 打包 → 打成**固定名** zip → 传到服务器 `releases/` 目录  
2. 在服务器上只改 `update_config.json` 里的 **version**（zip 名固定，不用改）  
3. 已安装的客户端启动约 2 秒后会请求 `check_update`，发现新版本会弹窗「是否立即更新」  
4. 用户点「立即更新」后，主程序退出，`updater.exe` 下载 zip 并覆盖安装目录，然后重新启动主程序  

---

## 一、你本机操作（每次发版）

### 1. 改版本号

在 `package.json` 里把 `"version"` 改成新版本，例如 `"1.0.1"`。

### 2. 打包

右键以管理员运行 `打包.bat`，得到 **`release\ShangpinCloudAssets\`** 目录（固定名，不会出现 win-unpacked）。

### 3. 打 zip 包（固定文件名）

把整个 **`release\ShangpinCloudAssets\`** 压缩成一个 zip，**每次都用同一个名字**：

- **`shangpin-cloud-assets.zip`**（固定，不要加版本号）

### 4. 把 zip 放到服务器

- 放到**公司主机上服务端**的 `releases` 目录里，**覆盖**原来的 `shangpin-cloud-assets.zip` 即可。  
- 服务端通过 `/releases/shangpin-cloud-assets.zip` 提供下载。

### 5. 在服务器上配置更新信息（只改版本号）

在**服务端运行目录**放或更新 `update_config.json`，**zip 文件名固定，只改 version**：

```json
{
  "version": "1.0.1",
  "fileName": "shangpin-cloud-assets.zip",
  "publicBaseUrl": "http://223.245.20.244:43123",
  "releaseNotes": "修复与优化（可选）"
}
```

- **version**：每次发版改成新版本号即可，客户端靠它判断是否有更新。  
- **fileName**：固定为 `shangpin-cloud-assets.zip`，不用改。  
- **publicBaseUrl**：你的公网地址，一般不用改。  
- **releaseNotes**：可选，会显示在更新弹窗里。

可参考项目里的 `update_config.example.json`。

---

## 二、客户端自动更新条件

- 客户端里的 `server_config.json` 配置了 `publicBaseUrl`（例如 `http://223.245.20.244:43123`），这样启动时才会去请求 `check_update`。  
- 客户端目录下有 `updater.exe`（与 `尚品易站云资产.exe` 同目录）。  
  - 若还没有：运行一次打包后，`release\ShangpinCloudAssets\` 里会自动带上 `updater.exe`（需本机已装 Python + PyInstaller）。  
- 外网能访问 `publicBaseUrl`（端口映射/防火墙已放行），否则外网客户端拿不到更新信息。

---

## 三、生成 updater.exe（若还没有）

在项目根目录：

```bash
pip install pyinstaller
pyinstaller updater.spec
```

生成文件在 `dist/updater.exe`。运行 `打包.bat` 时会自动复制到 `release\ShangpinCloudAssets\`，无需手动拷贝。

---

## 四、用户侧表现

- 已安装的客户端启动后约 2 秒，会请求 `http://你的公网地址/check_update`。  
- 若返回的 `version` 大于当前客户端版本，会弹窗「发现新版本 x.x.x，是否立即更新？」。  
- 用户点「立即更新」→ 主程序退出 → `updater.exe` 下载 zip 并解压覆盖当前安装目录 → 重新启动主程序，即完成自动更新。
